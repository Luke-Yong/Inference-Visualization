How this model learns

The same MoE network from the inference page is trained here with real hand‑written backpropagation (pure numpy — no PyTorch). It learns next‑token prediction: gradients of the cross‑entropy loss flow back through unembed → MoE → attention → embedding, and an optimizer nudges every weight to reduce the loss.

Tiny Shakespeare is real English tokenized per character, so the vocabulary is just the unique characters (~50) — small enough to keep the weight tables legible, while still being genuine English the model can learn structure from. Two standard training‑time choices: the whole sequence is processed at once with a causal mask (the KV cache is only an inference speed‑up), and the router uses soft routing (all experts) so it is differentiable; inference then keeps the top‑2.
Initializing all weights to 0 would prevent learning (symmetry & dead gradients), so we start from small random values and watch them change.
