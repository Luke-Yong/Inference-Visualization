# 大模型推理与训练可视化（MoE）

一个交互式的全栈网页应用，把 **专家混合（Mixture-of-Experts, MoE）Transformer**
的内部工作过程“看得见”——既包括 **推理** 前向过程（提示词 → 分词 → 注意力 →
专家 → 采样），也包括 **训练** 过程（前向 → 损失 → 反向传播 → 梯度更新）。整个
模型是一个 **真实但极小、且使用纯手写反向传播的网络**（纯 NumPy，不依赖 PyTorch）。

模型刻意做得很小，这样 *每一个* 中间张量都能以可读的表格/热力图展示出来。目标是
揭示现代大模型的 **运行机制**，而不是生成通顺的文本。

## 它能做什么

### 推理页面（`/`）
逐阶段地走完一次真实的前向过程：

1. **分词（Tokenization）** —— 提示词被切分为 token 并映射为整数 id。
2. **词嵌入（+ 位置编码）** —— 每个 token id 变成一个向量。
3. **带 KV 缓存的自注意力** —— Q/K/V 投影、各注意力头的打分、softmax 权重；新的
   K/V 会被 *追加到缓存* 中，从而不必重算历史 token（这正是解码很快的真正原因）。
4. **专家混合前馈网络（MoE FFN）** —— 一个 **路由器（router）** 为所有专家打分，
   只有 **4 选 2（top-2）** 的专家参与计算，其门控值重新归一化后再加权合并。
5. **输出与采样** —— logits → 温度（temperature）→ top-k → top-p（核采样）→
   采样出下一个 token。

多层模型会把 **每一层的“注意力 + MoE”模块** 依次堆叠展示。所有向量/矩阵都用
蓝/红热力图呈现。你可以单步/播放整个过程、调节采样参数、查看原始权重表，并勾选
**“使用训练后的权重（Use trained weights）”** 来运行你训练出的模型。

### 训练页面（`/train`）
实时观看同一个模型 **真正地学习**：

- 用 **真实的反向传播**（Adam/SGD）最小化交叉熵的“下一个 token 预测”损失，并通过
  **Server-Sent Events（SSE）流式传输**，因此 **损失曲线、进度条、生成样本会实时
  动态更新**。
- 拖动检查点（checkpoint）即可看到 **损失下降**、**下一个字符的准确率上升**、
  **词嵌入热力图** 从随机初始状态逐渐漂移，以及 **路由器/专家使用率出现分化**。
- 两个数据集：极小的 **词级语料**（表格最清晰易读）与 **字符级的 Tiny Shakespeare**
  （真实英文；首次运行时下载，本地缓存，并带有离线兜底文本）。
- 可调节数据集、优化器、学习率、训练轮数（epochs）、模型宽度（`d_model`）、
  深度（`layers`）、上下文窗口等。**“Reset to best”** 会恢复调好的最佳默认值；
  结果会 **按浏览器会话缓存**，因此在两个页面间来回切换不会重新训练。

## 它能帮助谁

- **学生与自学者**：用来建立对 Transformer/大模型实际计算过程的直觉——直接看到
  张量、注意力和路由，而不只是读公式。
- **教师/讲师**：需要一个可点击的实时教学素材，用来讲解注意力、KV 缓存、MoE 路由、
  温度/top-k/top-p 采样，以及前向/反向的训练循环。
- **初识 MoE 的工程师**：在接触生产框架前，先获得一个忠实而精简的“专家路由 +
  稀疏计算”心智模型。
- **任何好奇者**：想搞清楚“模型在逐 token 生成”底层到底意味着什么。

它 **不是** 生产级模型，也 **无法** 产出通顺文本——权重非常小。它是一个
教学/可视化工具。

## 它如何工作（架构）

```
浏览器 (HTML/CSS/JS)  ←→  Flask (app.py)  ←→  NumPy 模型 (engine.py / trainer.py)
   分步可视化界面          JSON + SSE              真实前向 + 反向传播
```

模型由 `num_layers` 个 Transformer-MoE 模块堆叠而成：

```
ids → 词嵌入 (+ 位置编码)
    → [ 自注意力（因果、多头、KV 缓存） + 残差
        → MoE 前馈（路由 → top-2 专家 → 加权求和） + 残差 ] × num_layers
    → 反嵌入 (unembed) → logits → 采样
```

- **专家（Experts）** 是结构完全相同的小型两层 MLP（`W1 → ReLU → W2`）；让它们彼此
  不同的是 **权重**——初始化时随机，随后在路由器门控的作用下 **通过训练实现专业化**。
- **训练期的简化**（均为标准做法，应用内也有说明）：训练时用因果掩码一次性处理整个
  序列（KV 缓存属于 *推理* 优化）；训练时路由采用 **软路由（soft routing）**（所有
  专家参与、可微），而推理时仅保留 **top-2**。
- 反向传播是 **手写并经过数值梯度校验** 的（不使用自动求导），因此在纯 NumPy 环境
  即可运行——包括尚无 PyTorch 轮子的 Python 3.14。

### 项目结构
| 文件 | 作用 |
|---|---|
| [`app.py`](./app.py) | Flask 服务：页面 + `/api/infer`、`/api/train_stream`（SSE）、`/api/status`、`/api/reset_weights` |
| [`engine.py`](./engine.py) | 推理模型：多层前向、KV 缓存、top-2 MoE、采样、完整分步轨迹、加载训练权重 |
| [`trainer.py`](./trainer.py) | 可训练模型 + 手写反向传播 + Adam/SGD；`train_stream` 生成器；保存权重与元信息 |
| [`dataset.py`](./dataset.py) | 词级语料 + 字符级 Tiny Shakespeare（下载/缓存 + 分词器） |
| `templates/`、`static/` | 前端（推理页与训练页、样式、可视化 JS） |
| `trained_weights.npz`、`trained_meta.json` | 训练后保存的模型（配置 + 词表 + 权重） |

## 快速开始

环境要求：**Python 3.10+** 与 `pip`。只需要 **Flask** 和 **NumPy**（无需 PyTorch）。

```bash
pip install -r requirements.txt
python app.py
```

然后打开 **http://127.0.0.1:5000**。

- `/` —— 推理可视化
- `/train` —— 训练可视化（首次加载会自动训练 Tiny Shakespeare；首次会下载约 1 MB
  文本，之后缓存到 `data/` 目录）

常见流程：打开 **/train** 观看训练（或点击 **Train model**），再打开 **/** 并勾选
**“Use trained weights”**，即可运行你刚训练出的模型。

> 调好的默认配置（d_model 32 · 2 层 · lr 0.03 · **170 轮**）在 Tiny Shakespeare
> 上可达到 **约 88% 的下一个字符准确率**，在开发服务器上约需 2–3 分钟；训练过程是
> 流式的（可实时观看进度），并按会话缓存。随时可点击 **“Reset to best”** 恢复这些
> 默认值。

## 说明与诚实的局限

- 输出 **并非** 通顺的文章——模型刻意做得极小以保证表格可读。即便下一个字符准确率
  达到约 88%，字符级的 Tiny Shakespeare 学到的也只是拼写/字母统计规律和“伪莎士比亚”
  片段，而非真正的句子（预测下一个 *字符* 远比生成有意义的文本容易）。
- 仅有 4 个极小专家、数据集也很小，因此专家 **专业化程度有限**；其 *机制* 是忠实的，
  但 *程度* 受限于规模。
- 内置的开发服务器仅供本地使用，不用于生产环境。
